## Points Clés

- **Redesign-v2 de l'Overscan EXR** — Intégration de l'implémentation validée du rendu d'overscan :
  - Les rendus EXR hors ligne respectent les fenêtres standards OpenEXR : `displayWindow = cadre de livraison` et `dataWindow = boîte englobante agrandie de l'overscan`.
  - Prise en charge complète pour l'output direct (OpenEXR, Multicouche) et les nœuds Compositor `File Output`.
  - Désactivation automatique des options d'overscan lorsque **Render Region** est activé.
  - Expose les propriétés Python RNA (`scene.cycles.overscan_mode`, `scene.cycles.overscan_size`, etc.).
  - Affiche un cadre de sécurité en superposition (safety frame overlay) dans la vue caméra.
- **Mise à jour des headers de kernel Deep EXR** — Restauration du fichier `deep_write.h` dans le package d'installation pour éviter l'échec de la compilation à la volée du noyau Cycles.
- **Nettoyage de l'interface utilisateur Deep EXR** — Retrait du panneau indépendant redondant Deep EXR tout en conservant le pipeline d'assemblage basé sur les nœuds File Output.
- **Isolation du cache de shaders OptiX** — Configuration d'un chemin par défaut propre pour le cache Nvidia OptiX (`%USERPROFILE%\AppData\Local\IndustrialCGPlatform\Cache\OptiX\`) afin de prévenir les conflits entre installations, tout en respectant l'override `OPTIX_CACHE_PATH` s'il est spécifié.
