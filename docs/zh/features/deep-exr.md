# Deep EXR 深度输出

<Badge type="tip" text="已发布" />

## 简介

Deep EXR 是一种渲染输出格式，在颜色数据旁边存储逐采样的深度信息。与只存储最近可见表面的标准"平面"EXR 文件不同，Deep EXR 保留了每个对最终像素有贡献的深度采样。这使得在 Nuke 等工具中进行**无损深度合成**成为可能，CG 层可以基于实际深度进行合并，而不是简单的叠加操作。

Industrial CG Platform 为 Blender Cycles 添加了原生 Deep EXR 输出支持。

## 为什么使用它

- **无损深度合并** — 在 Nuke 中使用 `DeepMerge` 合并多个 CG 渲染层，无边缘伪影。
- **逐采样深度** — 每个像素携带完整的深度信息。
- **体积支持** — 体积渲染数据包含在深度输出中。
- **合成器集成** — 同时支持直接场景输出和 Blender 合成器。

## 如何启用

### 场景输出

1. 打开 **属性 > 输出属性 > 输出**。
2. 将 **文件格式** 更改为 `Deep OpenEXR`。
3. 设置所需的 **Deep Tile Budget**。
4. 正常渲染场景。

### 合成器文件输出

1. 在合成器中添加 **文件输出** 节点。
2. 将其格式设置为 `Deep OpenEXR`。
3. 连接渲染层。

## 参数

| 参数 | 说明 | 默认值 |
| --- | --- | --- |
| **Deep Output** | 启用深度 EXR 输出格式 | 关闭 |
| **Deep Tile Budget** | 每个图块的深度采样内存预算 | 自动 |

## Nuke 工作流程

1. 使用 Deep EXR 输出渲染场景层。
2. 在 Nuke 中使用 `DeepRead` 节点导入 `.exr` 文件。
3. 使用 `DeepMerge` 基于深度合成层。
4. 使用 `DeepToImage` 将深度数据展平为标准图像。

## 已知限制

- **体积深度行为** — 内存密集型体积场景可能产生较大的深度文件。
- **内存使用** — 深度输出存储的数据远多于平面 EXR。使用 Deep Tile Budget 参数控制平衡。
