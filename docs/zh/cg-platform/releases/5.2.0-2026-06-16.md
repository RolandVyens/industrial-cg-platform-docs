# 5.2.0-2026-06-16

<Badge type="tip" text="最新版本" />

## 核心更新

- **EXR Overscan（溢画幅）Redesign-v2** — 交付经过完整验证的溢画幅渲染输出机制：
  - 离线 EXR 输出严格遵循规范：`displayWindow = 交付分辨率`，而扩展的 `dataWindow = 溢画幅画布分辨率`。
  - 完全支持直接 Cycles 渲染输出（OpenEXR、Multilayer）以及合成器中的 `File Output` 节点文件输出。
  - 当**渲染区域（Render Region）**激活时，自动锁定并停用溢画幅控制，确保管线安全性。
  - 暴露 Python RNA 属性接口（`scene.cycles.overscan_mode`, `scene.cycles.overscan_size` 等）。
  - 在视口相机视角内渲染半透明的安全框辅助线引导。
- **Deep EXR 内核头文件打包补齐** — 修复了缺少 `deep_write.h` 的打包遗漏，确保在独立运行包下 fallback 编译 Cycles 内核时不会发生编译中断。
- **Deep EXR 输出 UI 清理** — 移除了冗余的独立渲染属性面板，使 Deep 渲染与标准 Cycles File Output 节点合成工作流无缝融合。
- **OptiX 着色器缓存隔离** — 默认启用独立的产品着色器缓存路径（`%USERPROFILE%\AppData\Local\IndustrialCGPlatform\Cache\OptiX\`），避免与原生官方 Blender 缓存相互污染，同时保留对系统 `OPTIX_CACHE_PATH` 环境变量优先级的完全尊重。
